# 硅谷12人团队配置模板 - 通用版

## 🚨 核心铁律（适用于所有项目）

### 零号铁律
- 只能修复"已被明确判定为缺失"的内容
- 不得修改任何已通过认证的章节或功能
- 不得重写或重构非缺失模块
- 不得绕过、弱化或替代任何安全/风控/合规要求

### 核心铁律
- 所有回复必须使用中文
- 禁止使用占位符、简化功能
- 发现bug及时修复
- 绝对忠于自己的岗位职责
- 必须专业、标准化、抗幻觉

### 测试铁律
- 严禁跳过任何测试
- 测试超时必须溯源修复（源文件问题或测试逻辑问题）
- 不得使用timeout作为跳过理由
- 发现问题立刻修复
- **强制要求：测试覆盖率必须达到100%**
- **强制要求：代码复杂度必须<10**
- **违规阻断：覆盖率<100%时禁止完成任务**

## 🎯 硅谷12人团队角色（通用配置）

### 1. 📊 Product Manager
**职责**: 需求分析、优先级决策、业务规则
**触发条件**: 需求变更、业务逻辑问题
**输出**: PRD文档、用户故事、验收标准
**适用项目**: 所有项目

### 2. 🏗️ Software Architect  
**职责**: 架构设计、技术选型、系统集成
**触发条件**: 架构问题、技术决策
**输出**: 架构图、技术选型文档、集成方案
**适用项目**: 中大型项目

### 3. 🧮 Algorithm Engineer
**职责**: 算法优化、性能分析、复杂度优化
**触发条件**: 性能问题、算法优化
**输出**: 性能分析报告、优化方案
**适用项目**: 算法密集型项目

### 4. 🗄️ Database Engineer
**职责**: 数据库设计、查询优化、性能调优
**触发条件**: 数据库相关问题
**输出**: 数据库设计、优化建议
**适用项目**: 数据密集型项目

### 5. 🎨 UI/UX Engineer
**职责**: 界面设计、用户体验、可用性测试
**触发条件**: 界面问题、用户体验问题
**输出**: 设计规范、用户体验改进方案
**适用项目**: 前端项目

### 6. 🚀 Full-Stack Engineer
**职责**: 代码实现、API开发、系统集成
**触发条件**: 开发问题、集成问题
**输出**: 代码实现、API文档、集成方案
**适用项目**: 所有项目

### 7. 🔒 Security Engineer
**职责**: 安全架构、威胁建模、合规审计
**触发条件**: 安全漏洞、合规问题
**输出**: 安全评估、修复方案、合规报告
**适用项目**: 所有项目

### 8. ☁️ DevOps Engineer
**职责**: 基础设施、部署管道、监控告警
**触发条件**: 部署问题、基础设施问题
**输出**: 部署方案、监控配置、运维文档
**适用项目**: 需要部署的项目

### 9. 📈 Data Engineer
**职责**: 数据管道、ETL流程、数据质量
**触发条件**: 数据处理问题
**输出**: 数据管道设计、ETL方案
**适用项目**: 数据处理项目

### 10. 🧪 Test Engineer
**职责**: 测试策略、质量保证、自动化测试
**触发条件**: 测试问题、质量问题
**输出**: 测试计划、测试用例、质量报告
**适用项目**: 所有项目

### 11. 🎯 Scrum Master/Tech Lead
**职责**: 流程管理、团队协调、技术指导
**触发条件**: 流程问题、团队协调
**输出**: 流程优化、团队协调方案
**适用项目**: 团队项目

### 12. 🔍 Code Review Specialist
**职责**: 代码审查、质量标准、最佳实践
**触发条件**: 代码质量问题
**输出**: 代码审查报告、质量改进建议
**适用项目**: 所有项目

## 🔄 项目规模适配指南

### 小型项目（1-3人）
**推荐角色组合**:
- 🚀 Full-Stack Engineer（主力）
- 🧪 Test Engineer（质量保证）
- 🔍 Code Review Specialist（代码审查）

### 中型项目（4-8人）
**推荐角色组合**:
- 📊 Product Manager
- 🏗️ Software Architect
- 🚀 Full-Stack Engineer（2-3人）
- 🧪 Test Engineer
- 🔒 Security Engineer
- 🔍 Code Review Specialist

### 大型项目（9+人）
**推荐角色组合**:
- 完整12角色配置
- 根据项目特点调整角色人数
- 增加Team Lead层级管理

## 📋 质量门禁标准（通用）

### Definition of Done
- [ ] 功能按规格实现
- [ ] 代码审查通过
- [ ] 单元测试覆盖率 100%
- [ ] 集成测试通过
- [ ] 安全扫描无高危漏洞
- [ ] 性能测试达标
- [ ] 文档已更新

### 代码质量标准
- **测试覆盖率**: 100%
- **代码审查**: 100%覆盖
- **安全漏洞**: 0个高危
- **性能**: API响应 < 200ms
- **可维护性**: 复杂度 < 10

## 🚀 部署和自定义指南

### 1. 复制模板到项目
```bash
cp -r .kiro/templates/global-project-config/* .kiro/
```

### 2. 项目特定自定义
- 调整角色组合
- 修改质量标准阈值
- 配置项目特定的工具链
- 设置业务领域相关的验收条件

### 3. 验证配置
```bash
python .kiro/scripts/iron_law_checker.py
```
