**版本**: v1.6.0

## 🚀 快速开始（3步法）

### 步骤1: 告诉我你要实现什么功能

**示例**:
```
"我想实现遗传算法因子挖掘器（GeneticMiner）"
"我想实现数据清洗模块（DataSanitizer）"
"我想实现Soldier快脑决策系统"
```

### 步骤2: 我会自动做这些事

作为Kiro AI，我会自动：
1. ✅ 在白皮书中搜索该功能的定义
2. ✅ 在实现检查清单中找到对应的检查项
3. ✅ 确认功能的接口、参数、性能要求
4. ✅ 如果找不到定义，我会告诉你并拒绝实现

### 步骤3: 我会按照7步流程编码

1. 先写测试用例
2. 实现完整功能（无占位符）
3. 运行测试验证
4. 进行质量检查
5. 更新实现清单

**你不需要做任何准备工作，直接告诉我要实现什么即可！**

## 💬 实际对话示例

### 示例1: 实现新功能

**你说**:
```
"我想实现遗传算法因子挖掘器"
```

**我会做**:
1. 在白皮书中搜索 "GeneticMiner"
2. 找到第四章 4.1 的定义
3. 确认接口：`__init__`, `initialize_population`, `evaluate_fitness`, `evolve`
4. 确认性能要求（如果有）
5. 开始编写测试用例
6. 实现完整代码
7. 运行测试验证

**我会问你**:
- "我在白皮书第四章找到了GeneticMiner的定义，包含4个主要方法。我现在开始实现，对吗？"
- "测试文件放在 `tests/unit/chapter_4/test_genetic_miner.py`，可以吗？"

### 示例2: 功能不在白皮书中

**你说**:
```
"我想实现一个高级优化器（AdvancedOptimizer）"
```

**我会回复**:
```
抱歉，我在白皮书（00_核心文档/mia.md）中搜索了"AdvancedOptimizer"，
但没有找到相关定义。

根据MIA编码铁律1（白皮书至上），我不能实现未在白皮书中明确定义的功能。

建议：
1. 确认功能名称是否正确？可能是其他名字？
2. 如果是新功能，请先在白皮书中添加定义
3. 或者使用白皮书中已定义的类似功能

白皮书中定义的优化相关功能有：
- GeneticMiner（遗传算法）
- MetaEvolution（元进化）
- PromptEvolutionEngine（提示词进化）

您是指这些功能之一吗？
```

**你说**:

您希望我：
A. 一次性实现GeneticMiner的所有方法（推荐）
B. 先实现GeneticMiner，后续再实现其他类

请选择A或B？

## 🎯 最佳实践

### ✅ 推荐的做法

1. **明确需求**
   ```
   "我想实现白皮书第四章的GeneticMiner类"
   ```

2. **一次一个功能**
   ```
   "先实现GeneticMiner，完成后再实现Arena"
   ```

3. **信任流程**
   ```
   让我按照7步流程来，不要跳过测试
   ```

4. **及时反馈**
   ```
   如果代码有问题，告诉我具体哪里不对
   ```

### ❌ 不推荐的做法

1. **要求简化**
   ```
   ❌ "先写个框架，用TODO标记"
   ✅ "实现完整功能"
   ```

2. **跳过测试**
   ```
   ❌ "测试后面再写"
   ✅ "先写测试，再写代码"
   ```

3. **自创功能**
   ```
   ❌ "实现一个SuperOptimizer"
   ✅ "实现白皮书中定义的GeneticMiner"
   ```

4. **降低标准**
   ```
   ❌ "测试覆盖率70%就行"
   ✅ "测试覆盖率必须≥85%"
   ```

## 🔧 常用命令

### 搜索白皮书
```bash
# 搜索类名
grep -n "GeneticMiner" 00_核心文档/mia.md

# 搜索章节
grep -n "第四章" 00_核心文档/mia.md

# 搜索性能要求
grep -n "延迟" 00_核心文档/mia.md
```

### 查看实现进度
```bash
# 查看检查清单
cat 00_核心文档/IMPLEMENTATION_CHECKLIST.md

# 搜索特定功能
grep -n "GeneticMiner" 00_核心文档/IMPLEMENTATION_CHECKLIST.md
```

### 运行测试
```bash
# 运行单元测试
pytest tests/unit/chapter_4/test_genetic_miner.py -v

# 检查覆盖率
pytest tests/unit/chapter_4/ --cov=src/evolution --cov-report=term
```

### 质量检查
```bash
# 幻觉检查
python scripts/check_hallucination.py src/evolution/genetic_miner.py

# 代码质量检查
python scripts/pre_commit_check.py

# Pylint检查
pylint src/evolution/genetic_miner.py
```
